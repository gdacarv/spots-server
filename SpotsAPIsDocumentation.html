URL do servidor: http://matc89spots.appspot.com

Fetch Spots

Request

        GET /api/spots

        Parâmetros

        {latitude

longitude

limit(opcional)

offset(opcional)}

OU

{fromGroup (id)}

Retorno

        Lista de spots proximos ao ponto, limitados por “limit” e paginados por “offset” OU spots associados a algum group

        Exemplo

        /api/spots?latitude=-12.98735&longitude=-38.481527&limit=1

Result: [{“id”:12,

“name”:”Parque Costa Azul”,

“description”:”Parque que fica no Costa Azul”,

“sports”:[

{“id”:1,

“name”:”Patinação”},

{“id”:3,

“name”:”Ciclismo”}],

“latitude”:-12.998327,

“longitude”:-38.448293}]

Add/Edit Spots

Request

        POST /api/edit_spot

        Parâmetros

        id (opcional - Sem id indica novo spot, com id indica o Spot a ser editado)

        name

        description

        sports (lista de ids)

        latitude

        longitude        

creator (id do user criador do spot - apenas para Novo Spot)

Retorno

        Spot recém adicionado ou editado.

        Exemplo

        /api/edit_spot?name=Orla&description=Local de varios esportes&sports=4,3,23&latitude=-12.98735&longitude=-38.481527?creator=23

Result: {“id”:22,

“name”:”Orla”,

“description”:”Local de varios esportes”,

“sports”:[

{“id”:4,

“name”:”Corrida”},

{“id”:3,

“name”:”Ciclismo”},

{“id”:23,

“name”:”Basquete”}],

“latitude”:-12.98735,

“longitude”:-38.481527}

Fetch Groups

Request

        GET /api/groups

        Parâmetros

fromSpot(id - opcional*)

fromUser(id - opcional*)

*Pelo menos um desses parâmetros tem que ser fornecido

Retorno

        Lista de groups do user ou do spot

        Exemplo

Add/Edit Group

Request

        POST /api/edit_group

        Parâmetros

        id (opcional - Sem id indica novo group, com id indica o Group a ser editado)

        name

        description

        sport

        creator (id do user criador do group - apenas para Novo Group)

Retorno

        Group recém adicionado ou editado.

        Exemplo

        /api/edit_group?name=Radicais&description=Grupo de treino radical&sport=7&creator=23

Result: {“id”:9,

“name”:”Radicais”,

“description”:”Grupo de treino radical”,

“sport”:

{“id”:7,

“name”:”Slackline”},

“users”:[23],

“spots”:[]}

Fetch Users

Request

        GET /api/users

        Parâmetros

id(id - opcional*)

fromGroup(id - opcional*)

*Pelo menos um desses parâmetros tem que ser fornecido

Retorno

        Lista de users do group ou do user com id especificado

        Exemplo

        /api/users?fromGroup=9

Result: [{“id”:23,

“name”:”Gandalf Vader”,

“location”:”Salvador - BA”,

“facebookId”:”123456789”,

“googleplusId”:null,

“groups”:[9]}]

        Obs.: Requests passando fromGroup sempre resultam em lista [] e passando id sempre resulta em um único objeto {} (ver retorno de Add/Edit User).

Add/Edit User (Login)

        *Deve-se ser executado toda vez que o user logar no app

Request

        POST /api/edit_user

        Parâmetros

facebookId(id - opcional*)

googleplusId(id - opcional*)

name

location

*Pelo menos um desses parâmetros tem que ser fornecido

Retorno

        User recém logado. Aplicativo deve salvar essas informações, principalmente o Id, para fazer futuros requests.

        Exemplo

        /api/edit_user?facebookId=123456789&name=Gandalf Vader&location=Salvador - BA

Result: {“id”:23,

“name”:”Gandalf Vader”,

“location”:”Salvador - BA”,

“facebookId”:”123456789”,

“googleplusId”:null,

“groups”:[9]}

Entrar/Sair do Group

Request

        POST /api/inout_group

        Parâmetros

        userId

        groupId

        inOrOut (deve receber “in” se user está entrando no group e “out” se está saindo)

Retorno

        “OK” ou error.

        Exemplo

        /api/inout_group?userId=23&groupId=9&inOrOut=out

Result: { “result”:”OK”}

Link Group-Spot

Request

        POST /api/linkgroupspot

        Parâmetros

        spotId

        groupId

        linkOrUnlink(deve receber “link” se spot está sendo linkado ao group e “unlink” se está sendo deslinkado)

Retorno

        “OK” ou error.

        Exemplo

        /api/linkgroupspot?spotId=22&groupId=9&linkOrUnlink=link

Result: { “result”:”OK”}

Erros

        Todos os request pode resultar em erro caso parametros estejam incorretos ou houver algum erro no servidor. O modelo de erro é esse para todos os casos:

{ “error”:”Mensagem explicando o erro.”}
